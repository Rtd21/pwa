### P1: Fondasi (Wireless vs Mobile Computing)
Framework: PREP

Point: Perbedaan mendasar keduanya terletak pada media transmisi versus kemampuan perpindahan fisik pengguna.

Reason: Wireless merujuk pada komunikasi tanpa kabel fisik, sedangkan mobile merujuk pada perangkat yang dirancang untuk digunakan sambil berpindah tempat.

Example: Contoh hanya wireless adalah PC desktop menggunakan Wi-Fi (tidak pindah), hanya mobile adalah penggunaan laptop offline di kereta, dan keduanya adalah smartphone dengan paket data saat di bus.

Point: Dengan demikian, perangkat wireless tidak selalu mobile, dan perangkat mobile membutuhkan teknologi wireless untuk konektivitas saat bergerak.
### P2: Mobilitas (Hard vs Soft Handoff)

| Komponen | Penjelasan |
| :--- | :--- |
| **Alasan** | Handoff diperlukan untuk menjamin kelangsungan sesi komunikasi saat pengguna berpindah antar area cakupan BTS. |
| **Poin** | *Soft Handoff* jauh lebih baik untuk layanan *voice call* dibandingkan *Hard Handoff*. |
| **Bukti** | *Soft Handoff* menggunakan prinsip *make-before-break* (menyambung sebelum memutus), sedangkan *Hard Handoff* adalah *break-before-make*. |
| **Dampak** | Pada *Soft Handoff*, transisi suara mulus tanpa jeda, sedangkan *Hard Handoff* sering menyebabkan suara terputus sesaat atau *glitch*. |
| **Detail** | Keunggulan *Soft Handoff* memastikan tidak ada kehilangan data atau interupsi koneksi selama proses perpindahan sinyal terjadi. |

### P3: Jaringan (Standar IEEE & Mode Operasi)

| Komponen | Penjelasan |
| :--- | :--- |
| **Alasan** | Standarisasi diperlukan agar perangkat dari berbagai vendor dapat saling berkomunikasi dalam berbagai skala cakupan. |
| **Poin** | Terdapat empat kategori standar IEEE: **WPAN** (802.15), **WLAN** (802.11), **WMAN** (802.16), dan **WWAN** (802.20). |
| **Bukti** | Perbedaan utama adalah mode *Infrastructure* menggunakan Access Point pusat, sedangkan *Ad-Hoc* adalah komunikasi langsung antar perangkat. |
| **Dampak** | Mode *Infrastructure* memberikan stabilitas dan manajemen trafik yang lebih baik untuk jaringan skala besar dibandingkan *Ad-Hoc*. |
| **Detail** | Standar 802.11 (Wi-Fi) adalah yang paling umum diimplementasikan pada lapisan WLAN untuk kebutuhan akses nirkabel lokal. |

### P4: OS Mobile (Arsitektur & Sandboxing)
Framework: PREP

Point: Arsitektur mobile terdiri dari 4 lapisan (Kernel, Libraries/Runtime, Framework, Application) dengan sistem keamanan sandboxing yang ketat.

Reason: Sandboxing di mobile lebih ketat daripada desktop untuk mengisolasi setiap aplikasi agar tidak bisa mengakses data aplikasi lain atau sistem inti tanpa izin.

Example: Di Android, setiap aplikasi berjalan dengan User ID uniknya sendiri; jika satu aplikasi terkena malware, ia tetap terkurung dalam "sandbox" tersebut dan tidak merusak sistem.

Point: Hal ini menjadikan integritas sistem operasi mobile jauh lebih aman terhadap serangan eksternal dibandingkan sistem desktop konvensional.

### P5: OS Lanjutan (LMK & Intent)

| Komponen | Penjelasan |
| :--- | :--- |
| **Alasan** | Sumber daya mobile (RAM dan baterai) sangat terbatas, sehingga OS harus mampu memprioritaskan proses secara cerdas. |
| **Poin** | *Low Memory Killer* (LMK) akan menghentikan proses berdasarkan hierarki, mulai dari yang paling tidak terlihat oleh pengguna. |
| **Bukti** | *Intent Eksplisit* langsung menyebut nama komponen target, sedangkan *Intent Implisit* hanya menyebutkan aksi (misal: "share"). |
| **Dampak** | Dengan LMK, aplikasi *foreground* tetap lancar meskipun banyak aplikasi lain terbuka, karena OS otomatis membersihkan RAM. |
| **Detail** | *Intent* memungkinkan interaksi antar aplikasi yang aman tanpa harus saling mengetahui struktur kode internal masing-masing. |
Penjelasan (Framework PREP)
Untuk melengkapi tabel di atas, berikut adalah penjelasan naratif mengenai bagaimana komponen ini bekerja secara sinergis dalam sebuah sistem operasi mobile:

Point: Sistem Operasi mobile menggunakan manajemen proses yang sangat agresif untuk menjaga stabilitas perangkat.

Reason: Hal ini diperlukan karena keterbatasan sumber daya fisik (RAM) memaksa OS harus bertindak sebagai pengatur lalu lintas yang memprioritaskan aplikasi yang sedang berinteraksi langsung dengan pengguna (foreground).

Example: Ketika kamu sedang membuka aplikasi kamera yang berat, LMK mungkin akan membunuh proses aplikasi media sosial yang ada di background agar kamera mendapatkan alokasi RAM yang cukup dan tidak lag.

Point: Dengan demikian, kombinasi antara sistem Intent untuk komunikasi dan LMK untuk manajemen memori memastikan ekosistem aplikasi berjalan efisien dan aman bagi pengguna.

### P6: Mobile Web (Strategi Native vs Hybrid vs PWA)
Framework: PREP

Point: Untuk klien dengan budget terbatas dan kebutuhan 2 platform, strategi yang direkomendasikan adalah PWA (Progressive Web App).

Reason: PWA lebih murah karena hanya butuh satu tim web untuk semua platform dan tidak memerlukan pembagian keuntungan dengan App Store.

Example: Dibandingkan membangun Native yang mahal karena butuh tim Swift dan Kotlin, PWA memungkinkan aplikasi inventaris tetap berjalan meski sinyal buruk (Offline-First).

Point: Kesimpulannya, PWA memberikan efisiensi biaya terbaik tanpa mengorbankan fungsionalitas dasar aplikasi bisnis multi-platform.
