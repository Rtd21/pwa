Berdasarkan foto papan tulis yang Anda bagikan, sepertinya Anda sedang mempelajari perbandingan berbagai arsitektur pengembangan aplikasi mobile. Fokus utamanya adalah mengevaluasi kelebihan dan kekurangan antara aplikasi Native, Hybrid, dan PWA (Progressive Web App).
Berikut adalah penjelasan mendalam mengenai perbandingan sistem tersebut:

1. Analisis Arsitektur Aplikasi Mobile
Dosen Anda membagi teknologi mobile menjadi tiga pilar utama. Berikut adalah perincian karakteristiknya:
A. Native App (Performa Maksimal)
Aplikasi yang dibangun khusus untuk platform tertentu (Java/Kotlin untuk Android, Swift/Objective-C untuk iOS).
- Akses Hardware: Memiliki akses Full ke sensor perangkat (NFC, Kamera, Bluetooth) tanpa perantara.
- Performa: Paling tinggi (High) karena kode berjalan langsung di atas sistem operasi. Sangat disarankan untuk aplikasi berat seperti AR/VR atau Game.
- Investasi: Biaya dan kebutuhan tim sangat tinggi karena Anda harus membuat dua basis kode yang berbeda untuk Android dan iOS.

B. Hybrid App (Jalan Tengah)
Aplikasi web yang "dibungkus" dalam kontainer native. Menggunakan framework seperti React Native, Flutter, atau Ionic (Capacitor).
- Akses Hardware: Menggunakan Plugin untuk menjembatani kode web dengan fungsi hardware.
- Performa: Berada di level Medium. Cukup untuk aplikasi bisnis atau e-commerce, namun tidak secepat native.
- Efisiensi: Lebih hemat biaya karena satu tim bisa mengembangkan aplikasi untuk dua platform sekaligus.

C. PWA - Progressive Web App (Efisiensi Web)
Aplikasi web yang memiliki kemampuan menyerupai aplikasi native namun berjalan di browser.
- Akses Hardware: Terbatas pada Web Standard API. Belum bisa mengakses fitur hardware yang sangat spesifik atau mendalam.
- Performa: Sangat bergantung pada optimasi JavaScript (Service Workers) dan kecepatan browser.
- Distribusi: Tidak perlu lewat Play Store/App Store. Cukup melalui URL/Web, sehingga biaya distribusinya sangat rendah (Low).

2. Penerapan Framework PREP dalam Mobile Computing
Untuk menjelaskan secara komprehensif, Anda bisa menggunakan kerangka PREP yang ada di pojok kiri papan tulis tersebut untuk memberikan argumen pemilihan teknologi:
- P (Point): Nyatakan teknologi mana yang dipilih.
Contoh: "Untuk proyek ini, kita sebaiknya menggunakan Native App."
- R (Reason): Berikan alasan teknis berdasarkan dimensi perbandingan (Akses hardware, performa, atau biaya).
Contoh: "Sebab aplikasi ini membutuhkan pemrosesan grafis AR/VR yang intensif dan akses sensor kamera yang sangat responsif."
- E (Example): Berikan contoh nyata atau skenario penggunaan.
Contoh: "Aplikasi seperti Pokémon GO memerlukan integrasi sensor GPS dan kamera yang sangat stabil yang hanya bisa dicapai maksimal oleh Native."
- P (Point): Tegaskan kembali kesimpulan Anda.
Contoh: "Oleh karena itu, meskipun biayanya tinggi, Native adalah pilihan paling aman untuk menjamin pengalaman pengguna yang lancar."
