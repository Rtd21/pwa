1. Akses Hardware (Hardware Access)
Point: Native memiliki akses perangkat keras yang paling superior dan langsung.

Reason: Karena ditulis dengan bahasa pemrograman spesifik platform, ia bisa mengakses seluruh fitur sensor tanpa perantara.

Example: Di papan tulis tertulis "Full NFC, Cam" untuk Native. Sebaliknya, Hybrid menggunakan "Plugin" (seperti Capacitor atau React Native) untuk menjembatani kode web ke hardware, sementara PWA terbatas pada "Web Standard API".

Point: Jadi, jika aplikasi sangat bergantung pada fitur hardware yang kompleks (seperti sensor medis atau kamera canggih), Native adalah pilihan mutlak.

2. Performa (Performance)
Point: Native menawarkan performa paling tinggi dibandingkan metode lainnya.

Reason: Kode dieksekusi langsung oleh sistem operasi tanpa lapisan tambahan, sehingga lebih efisien dan responsif.

Example: Papan tulis mencatat Native dengan status "Height" (maksudnya High/Tinggi), Hybrid di level "M" (Medium), dan PWA menggunakan "Service Worker" untuk membantu performa di web.

Point: Untuk aplikasi dengan beban grafis berat seperti game atau pengolahan data real-time, Native tetap yang terbaik.

3. Biaya & Tim (Cost & Team)
Point: PWA adalah yang paling efisien dari segi biaya dan kebutuhan tim.

Reason: Kamu hanya perlu satu tim web developer untuk membuat aplikasi yang bisa diakses di semua perangkat melalui browser.

Example: Di papan tulis disebutkan Native membutuhkan "3 Tim (OS)" (misal Android, iOS, Windows) yang berarti biaya tinggi (High). Hybrid lebih hemat karena "2x (P + W)" (biasanya satu basis kode untuk Android & iOS), sedangkan PWA diberi label "Low".

Point: PWA sangat cocok untuk bisnis yang ingin menghemat biaya operasional namun tetap memiliki kehadiran di perangkat mobile.

4. Distribusi (Distribution)
Point: Metode distribusi menentukan bagaimana pengguna menemukan aplikasi kamu.

Reason: Native dan Hybrid harus melalui proses kurasi di toko aplikasi resmi, sementara PWA berbasis web.

Example: Native didistribusikan melalui "Playstore, AppStore". Hybrid juga masuk ke store. Sedangkan PWA didistribusikan via "Web" (cukup buka URL dan "Add to Home Screen").

Point: Jika ingin menghindari biaya bagi hasil atau proses review toko aplikasi yang ketat, PWA memberikan kebebasan lebih.

5. Pilihan Tepat (Use Case)
Point: Pemilihan teknologi harus didasarkan pada tujuan akhir produk.

Reason: Tidak ada teknologi yang sempurna untuk semua hal; semuanya bergantung pada fungsionalitas utama.

Example: Catatan menyebutkan Native cocok untuk "Game AR/VR". Hybrid cocok untuk aplikasi seperti "App Logistik, ERP, POS" (Point of Sale). PWA cocok untuk konten yang butuh akses cepat via web (Web/SEO).

Point: Gunakan Native untuk pengalaman user yang imersif, Hybrid untuk aplikasi bisnis multi-platform, dan PWA untuk aksesibilitas luas tanpa instalasi rumit.
